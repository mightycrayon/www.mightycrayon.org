/ Frequently Asked Questions

  1. What is Fleava's technology Stack?

    +

    We typically use Javascript frameworks for web development. We use latest and popular technology stacks such as Node JS, Vue JS, React and Nuxt. We typically use AWS S3 for the CDN and MongoDB for the database.

    For mobile apps, we can go hybrid with React Native or Flutter, and we can go with native technology such as Swift.

  2. What is a CMS?

    +

    A content management system (CMS), An application used to manage material enables numerous contributors to generate, update, and publish information. As with a website, a CMS often uses a database to store content and a presentation layer to display it.

  3. Which CMS does Fleava use?

    +

    We develop and use our very own Content Management System: Vold. Vold is a modular, headless CMS based on NodeJS, and is loved by our customers because of its human-centric approach.

    However, we also use general CMS such as WordPress at times, if you require some specific WordPress plugins for the site.

  4. Should I use Hybrid or Native technology for my App?

    +

    Use Hybrid if you have fewer than four months to develop an app and you want to evaluate its viability in a small private market. Hybrid apps require less upkeep and are quicker and easier to design than native apps.

    Your hybrid app's performance is wholly dependent on how quickly the user's browser loads pages. You may create hybrid apps from a single foundation, enabling you to update different versions of your app with new features.

    If the test is successful, switch to native. You've saved yourself time and money if it doesn't work.

    Read: Developing a Mobile App for your Business

  5. Do you do design-only service?

    +

    Yes, we do offer standalone services, such as UX research, UI design and Frontend Development.

    Read: Why good web design is a must

  6. How long does it take to develop a Website?

    +

    A unique website must be created from scratch, which takes time. If you're considering a significant project with a realistic budget, you may be considering a timeline of several weeks or months. That's normal, but typically, building a website takes four months on average.

    It's crucial to specify the kind of websites you're referring to while discussing web development. A straightforward framework for categorization helps to grasp this since the scope, complexity, and talents required to construct websites can vary greatly.
